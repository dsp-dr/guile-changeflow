#+TITLE: ChangeFlow Specification
#+STARTUP: overview

ITIL 4 compliant change management system with MCP integration, built in GNU Guile and deployed as a Cloudflare Worker.

* Goals

- Implement full ITIL 4 change lifecycle: RFC submission through implementation and closure
- Automated risk assessment with scoring, freeze period enforcement, and deployment window calculation
- Expose change management tools via Model Context Protocol (MCP) for AI agent integration
- CAB (Change Advisory Board) approval workflows with audit trails
- Chaos engineering simulation for testing change management resilience

* Non-Goals

- General-purpose project management or ticketing
- Real-time monitoring or alerting (observability is external)
- User authentication management (delegated to GitHub OAuth via Cloudflare)

* Architecture

** Dual Runtime

The system has two runtimes that share the same ITIL domain model:

| Runtime | Purpose | Location |
|---------+---------+----------|
| GNU Guile 3.x | Core domain logic, risk engine, simulator, web dashboard | =src/= |
| Node.js | MCP server (local dev + npm package), Cloudflare Worker (production) | =bin/=, =infra/cloudflare/= |

| =models/approval.scm= | CAB approval workflow |
| =models/audit.scm= | Audit trail recording |
| =risk/calculator.scm= | Risk scoring engine (0-100 scale) |

| =simulator/= | Chaos engineering: change generation, failure injection, deployment pipeline |
| =web/= | HTTP dashboard, executive view, REST API |

| =integrations/= | GitHub, notifications, routing |

| Component | Role |

| Tool | Description |
|------+-------------|

** State Machine

#+BEGIN_SRC
submitted -> assessing -> approved -> implementing -> completed
                       -> rejected                 -> failed -> implementing
                       -> needs-info -> assessing
          -> cancelled (from any non-terminal state)
#+END_SRC

Terminal states: =completed=, =rejected=, =cancelled=.

** Database

SQLite with WAL mode. Schema in =migrations/001_initial_schema.sql=.

Core tables: =change_requests=, =users=, =approvals=, =state_transitions=,
=audit_log=, =risk_assessments=, =comments=, =dependencies=, =affected_systems=, =notifications=.

* Key Design Decisions

- *Dual runtime*: Guile for domain logic purity and Scheme expressiveness; Node.js/Cloudflare for production MCP hosting with OAuth
- *ITIL 4 strict compliance*: State transitions are validated against a fixed map; no shortcut paths
- *Risk scoring 0-100*: Composite of keyword analysis, system count, time-of-day, and freeze period factors
- *MCP over JSON-RPC 2.0*: Standard protocol for AI agent tool discovery and invocation
- *GitHub OAuth*: Production access gated through GitHub OAuth flow via Cloudflare Worker

* Known Limitations

- No LICENSE file committed to the repository
- SQLite storage is local; no shared persistence across Cloudflare Worker invocations (uses KV when available)
